## ADDED Requirements

### Requirement: 商品一覧を表示する
システムは `/products` で microCMS から取得した全商品をカード形式で表示しなければならない (SHALL)。
各カードには商品画像・商品名・カテゴリ・総合おすすめ度（星）を含む。

#### Scenario: ユーザーが商品一覧ページを開く
- **WHEN** ユーザーが `/products` にアクセスする
- **THEN** システムは登録済み商品の一覧をカードグリッドで表示する

#### Scenario: 商品が存在しない場合
- **WHEN** microCMS に商品データが 0 件の状態で `/products` にアクセスする
- **THEN** システムは「商品が登録されていません」等の空状態メッセージを表示する

### Requirement: カテゴリでフィルターできる
システムはカテゴリ（アイス / チョコレート / ドリンク / ケーキ・焼き菓子）で商品を絞り込めなければならない (SHALL)。
フィルター状態は URL クエリパラメータ (`?category=ice` など) に反映する。

#### Scenario: カテゴリを選択してフィルターする
- **WHEN** ユーザーが「アイス」フィルターを選択する
- **THEN** システムはカテゴリが「アイス」の商品のみ表示し、URL を `?category=ice` に更新する

#### Scenario: フィルターを解除する
- **WHEN** ユーザーが「すべて」フィルターを選択する
- **THEN** システムはすべての商品を表示する

### Requirement: 商品をソートできる
システムは「総合おすすめ度順（高→低）」「新着順」でソートできなければならない (SHALL)。

#### Scenario: おすすめ度順にソートする
- **WHEN** ユーザーが「おすすめ度順」を選択する
- **THEN** システムは総合おすすめ度の高い順に商品を並べ替えて表示する

### Requirement: 商品詳細へ遷移できる
システムは商品カードをクリックすると `/products/[id]` に遷移しなければならない (SHALL)。

#### Scenario: 商品カードをクリックする
- **WHEN** ユーザーが商品カードをクリックする
- **THEN** システムは対応する商品詳細ページ (`/products/[id]`) に遷移する
